# logout

退出登录

## 功能介绍

:::info 说明

退出登录回调成功后，UI 返回登录界面，等待用户输入账号密码。  
如果切换账号，需要等待用户 A 的 logout 回调成功后才能再次调用用户 B 的 login，否则 login 可能会失败。  
普通情况下，如果您的应用生命周期跟 IMSDK 生命周期一致，退出应用前可以不登出，直接退出即可。  
登出成功后，不会再收到其他人发送的新消息。

:::

### 函数原型

```ts showLineNumbers

IMSDK.logout(): Promise<WsResponse>;

```

### 输入参数

无

### 返回结果

| 参数名称        | 参数类型                                                      | 描述         |
| --------------- | ------------------------------------------------------------- | ------------ |
| Promise.then()  | Promise<[WsResponse](docs/sdks/Class/Response/Response.mdx)\> | 调用成功回调 |
| Promise.catch() | Promise<[WsResponse](docs/sdks/Class/Response/Response.mdx)\> | 调用失败回调 |

### 代码示例

```js showLineNumbers
import { getSDK } from 'open-im-sdk-wasm';
const IMSDK = getSDK();

const status = IMSDK.logout()
  .then(() => {
    // 退出登录成功
  })
  .catch(({ errCode, errMsg }) => {
    // 调用失败
  });
```

<TabItem value="uni-app">

### 函数原型

```ts showLineNumbers

IMSDK.asyncApi('logout',operationID: string): Promise<void>;

```

### 输入参数

| 参数名称    | 参数类型 | 是否必填 | 描述                                                    |
| ----------- | -------- | -------- | ------------------------------------------------------- |
| operationID | string   | 是       | 操作 ID，用于定位问题，保持唯一，建议用当前时间和随机数 |

### 返回结果

> 通过`openim-uniapp-polyfill`包将函数 Promise 化，调用时需要使用`then`和`catch`判断并处理成功和失败回调。

| 参数名称        | 参数类型                                                         | 描述         |
| --------------- | ---------------------------------------------------------------- | ------------ |
| Promise.then()  | Promise<void\>                                                   | 调用成功回调 |
| Promise.catch() | Promise<[CatchResponse](docs/sdks/Class/Response/Response.mdx)\> | 调用失败回调 |

### 代码示例

```js showLineNumbers
import IMSDK from 'openim-uniapp-polyfill';

IMSDK.asyncApi('logout', Date.now())
  .then(() => {
    // 退出登录成功
  })
  .catch(({ errCode, errMsg }) => {
    // 调用失败
  });
```
