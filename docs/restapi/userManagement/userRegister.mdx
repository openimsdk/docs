## 用户注册

</center>

### 简要描述

- 一般是通过业务服务器（如 chat 服务器）完成用户的账号注册后，再注册到 IM 系统中，这样业务系统和 IM 系统完成账号打通。

#### 接口状态

- 已完成

### 请求方式

- `post`

### 请求 URL

### 请求参数示例

```json
{
  "secret": "openIM123",
  "users": [
    {
      "userID": "kernaltestuid2",
      "nickname": "yourNickname",
      "faceURL": "yourFaceURL"
    }
  ]
}
```

| 字段名         | 选填 | 类型   | 说明        |
| :------------- | :--- | :----- | ----------- |
| secret         | 必填 | string | openIM 密钥 |
| users          | 必填 | array  | 用户列表    |
| users.userID   | 必填 | string | 用户 ID     |
| users.nickname | 必填 | string | 用户昵称    |
| users.faceURL  | 必填 | string | 用户头像    |

### 成功返回示例

```json

  "errCode": 1001,
  "errMsg": "ArgsError",
  "errDlt": ""
```

### 失败返回示例的参数说明

| 参数名  | 类型   | 说明                          |
| :------ | :----- | :---------------------------- |
| errCode | int    | 错误码,具体查看全局错误码文档 |
| errMsg  | string | 错误简要信息                  |
| errDlt  | errDlt | 错误详细信息                  |
