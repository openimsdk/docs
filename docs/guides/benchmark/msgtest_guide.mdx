# msgtest 使用说明（压力/容量）

msgtest 用于制造大量在线连接与消息吞吐压力，配合 integration_test 的抽样校验，形成“压力 + 可靠性/时延”的组合测试。

- 程序路径：`open-im-sdk-core-enterprise/msgtest/`
- 入口：`msgtest/main/main.go`

---

## 快速开始

1）进入目录并编译：

```bash
cd open-im-sdk-core-enterprise/msgtest

go build -o main ./main
```

2）修改连接配置（必须）：

- 生效位置：`open-im-sdk-core-enterprise/msgtest/module/pressure.go`
- 需要修改的变量：
  - `TESTIP`（服务端 IP 或域名）
  - `APIADDR`（HTTP API 地址，默认 `http://<TESTIP>:10002`）
  - `WSADDR`（WebSocket 地址，默认 `ws://<TESTIP>:10001`）
  - `MANAGERUSERID`（管理员账号，用于获取 token 等管理操作）
  - `SECRET`（管理员密码/密钥）
  - `PLATFORMID`（平台 ID，例如 Android/Windows）

3）启动压测（示例见下文“示例命令”）。

---

## 压测前准备

### 服务端与客户端数据清理

建议每次压测前保证环境干净，否则会影响会话数量、seq、消息统计等口径。

- 服务端：按部署方式清理数据并重启。
- 客户端：清理 msgtest 的本地数据目录（主流程里 SDK 初始化使用的 `DataDir` 为 `./`）。

## 参数说明

| 参数 | 含义 | 类型 | 默认值 |
| --- | --- | --- | --- |
| `-o` | 在线用户总数 | int | 20000 |
| `-s` | 起始用户（与 `-e` 配合使用） | int | 0 |
| `-e` | 结束用户（与 `-s` 配合使用） | int | 0 |
| `-sr` | 单聊抽样率 | float | 0.01 |
| `-rs` | 单聊随机发送者数量 | int | 0 |
| `-rr` | 单聊每个发送者随机接收者数量 | int | 0 |
| `-c` | 每个发送者向每个接收者发送的消息条数 | int | 0 |
| `-i` | 每个发送者发送间隔（毫秒） | int | 1000 |
| `-r` | 是否注册用户 | bool | false |
| `-u` | 仅保持在线，不发送消息 | bool | false |
| `-pp` | 是否开启 pprof（0.0.0.0:6060） | bool | false |
| `-gsr` | 群聊发送者比例 | float | 0.1 |
| `-gor` | 群聊在线比例 | float | 0.0 |
| `-htg` | 10 万人群数量 | int | 0 |
| `-ttg` | 1 万人群数量 | int | 0 |
| `-otg` | 1 千人群数量 | int | 0 |
| `-hog` | 100 人群数量 | int | 0 |
| `-fog` | 50 人群数量 | int | 0 |
| `-teg` | 10 人群数量 | int | 0 |

说明：Go 的 bool 参数使用 `-u`、`-r` 表示 true，或使用 `-u=true`、`-r=true`。

## 示例命令

- 5 万在线（仅保持在线）：

```bash
nohup ./main -o 50000 -u >> openIM.log 2>&1 &
```

- 5 万在线 + 单聊压测（示例：300 发送者，300 接收者，每对 100 条）：

```bash
nohup ./main -o 50000 -rs 300 -rr 300 -c 100 -i 1 -r >> openIM.log 2>&1 &
```

- 10 万在线（双机，各 5 万，仅保持在线）：

```bash
# 客户端 A
nohup ./main -o 50000 -u >> openIM.log 2>&1 &

# 客户端 B
nohup ./main -s 50000 -e 100000 -u >> openIM.log 2>&1 &
```

## 注意事项

### 动态端口范围（客户端压测机）

当长连接数量较大时，客户端可能因本地端口耗尽导致连接异常。建议按需调整动态端口范围：

```sh
sudo sysctl -w net.ipv4.ip_local_port_range="10000 65000"
```

原因：保证有足够的动态端口数模拟长连接。
